# 技能：系統技能生成與擴充

## 1. 你的職責
你負責協助使用者建立新的 Agent 技能。因為安全與穩定性考量，**你不能直接寫入檔案**。你必須將規劃好的內容，使用 Markdown 程式碼區塊（Code Block）輸出在對話中，讓使用者自行複製貼上。

## 2. 執行步驟 (SOP)
1. **需求訪談**：向使用者確認新技能的目標、何時該觸發、以及預期的輸出是什麼。
2. **參考現有架構 (可選)**：如果有需要，呼叫 `read_workspace_code` 看看使用者現有的工具是怎麼寫的，以保持風格一致。
3. **產出結構草案**：
   在回覆中，請提供以下三個部分：
   - **資料夾名稱**：建議一個全小寫、底線分隔的名稱（例如：`water_tracker`）。
   - **SKILL.md 草稿**：包含 `name`、`description` (Frontmatter)，以及具體的 SOP 步驟。
   - **TypeScript 模板草稿**：提供一個包含 `zod` Schema 與 `@langchain/core/tools` 的 `.ts` 基本骨架，讓使用者知道要在哪裡填寫商業邏輯。

## 3. 輸出規範
請務必將草稿包裝在 Markdown 的代碼區塊中，方便使用者一鍵複製。撰寫 SKILL.md 時，請多解釋「為什麼」要這樣做，而不是給予死板的格式限制。
