# 系統技能與工具索引 (SKILLS_INDEX)
這是系統目前具備的所有技能與對應工具清單。當你需要執行任務時，請參考以下索引呼叫對應工具：

## 1. 視覺與營養分析管線
- **場景**：使用者上傳食物圖片，或要求計算熱量。
- **可用工具**：
  - `analyze_food_image`: 讀取圖片路徑，辨識食材與預估重量 (回傳 JSON)。
  - `calculate_nutrition`: 接收食材與重量，計算精確的三大營養素與總熱量。

## 2. 系統知識庫管理
- **場景**：需要確認系統目前的總體飲食規則，或使用者要求新增/修改通用規則。
- **可用工具**：
  - `read_knowledge_tool`: 讀取目前的營養學規則 (NUTRITION_RULES)。
  - `update_knowledge_tool`: 覆寫更新通用規則 (需人類審核)。

## 3. 使用者背景檢索與存檔 (資料庫連線)
- **場景**：提供建議前需了解使用者背景，或分析完成後需要將這餐的紀錄寫入雲端資料庫。
- **可用工具**：
  - `get_user_profile`: 讀取使用者的健康背景（體徵、忌口 taboo、疾病史 disease），確保建議不會產生健康風險。
  - `get_chat_history`: 讀取該聊天室近期的對話與熱量紀錄，了解今日攝取狀況。
  - `log_diet_history`: 當營養分析與建議都完成後，將對話與熱量數據寫入 Supabase 永久保存。

## 4. 動態特徵學習 (Memory Summarizer)
- **場景**：在對話中發現使用者主動提及新的飲食禁忌（例如：對海鮮過敏、開始吃素）或疾病史（例如：檢查出高尿酸）。
- **可用工具**：
  - `update_user_profile`: 立刻將使用者的新特徵寫入資料庫，以優化未來的個人化建議。

## 5. 網頁資訊查證 (Web Search)
- **場景**：使用者詢問特定飲食法、健康新聞，或是要求查證衛福部/FDA 等權威機構的資訊。
- **可用工具**：
  - `internet_search`: 透過 Tavily 連上網際網路搜尋最新資訊。搜尋後請務必以「### 查證結果」為標題，為使用者總結論點。
